The Premise

DISSOCIATED is a first-person psychological detective game developed in Unreal Engine 5.

The player investigates a missing person case inside a confined residential space. Progression is driven by observation, collection of clues, and structured deduction rather than combat or scripted events.

The core mechanic revolves around ‚ÄúThreads‚Äù - investigative chains composed of interconnected clues. Completing a thread alters the environment and advances the narrative.

My Role: Solo Developer - Game Design, Blueprint Programming, UI/UX, Level Design, Narrative Direction.

Core Gameplay Framework

The gameplay loop is built around five primary systems:

  • Exploration: Free movement (jump, crouch, flashlight control) within a restricted environment designed for repeated traversal.
  • Interaction: Context-based object interaction system (E key) triggering inspections, item pickups, or narrative events.
  • Inventory: Array-based item system accessible via (I), supporting item usage and dependency logic.
  • Deduction Log: Thread-based investigation interface (B) that tracks collected clues and validates progression.
  • State Progression: Environmental changes triggered by completed threads or item use.

This structure ensures that narrative progression is system-driven rather than scripted.

Systems Architecture (Blueprint)

The project is built entirely using Blueprint-based modular systems.

  • Item Structs: Each item contains ID, icon, description, dependency flags, and usable targets.
  • Thread System: Data-driven deduction validation system that checks required clue states before progression.
  • State Controller: Centralized logic manager that updates environment variables when threads are completed.
  • UI-State Separation: Game logic runs independently of UI widgets to avoid freezing gameplay states.

Systems were designed to be scalable, allowing new items or threads to be added without restructuring the entire project.

Level Design & Architecture

Leveraging my background in Architectural Design, I created environments that tell a story through space.

  • Assets: A hybrid pipeline using custom Blender models for key narrative items and Fab/Quixel assets for environmental textures.
  • Audio: Implemented royalty-free soundscapes (Pixabay) using Sound Cues to create spatial tension.

Game Logic & Flow Design

The investigation system is structured as conditional logic chains rather than linear scripted events. Player actions activate flags that validate deduction threads, which then trigger controlled environmental changes.

Each ‚ÄúThread‚Äù contains required clue states. Once validated, the system updates global variables that modify level states, unlock areas, or introduce new narrative elements.

Flow diagrams were used during development to ensure clarity between player input, system validation, and environmental response. This prevented circular logic and ensured scalable progression design.

  • Thread Validation Flow: Clue Collected ‚Üí Flag Activated ‚Üí Thread Condition Check ‚Üí Environment Update
  • Item Logic Flow: Inventory Select ‚Üí Condition Check ‚Üí Target Validation ‚Üí Event Trigger
  • UI State Flow: Gameplay ‚Üí UI Overlay ‚Üí Safe Pause State ‚Üí Resume Without Logic Break

UI & Interaction Design

The UI was designed to support immersion while maintaining clarity and system transparency. All interface elements were built using Unreal Engine's UMG framework.

  • Custom HUD: Minimal investigative overlay designed to avoid screen clutter.
  • Inventory System (I): Grid-based array system displaying item icons with usable state validation.
  • Deduction Log (B): Structured thread interface showing collected clues and completion states.
  • Pause System (P): Controlled game state handling without interrupting underlying logic variables.
  • Flashlight System (F): Toggle-based visibility modifier impacting atmosphere and exploration.

UI logic is separated from gameplay logic to ensure safe state transitions and prevent unintended execution during overlays.

Character Design - The Detective

The player controls Detective Adrien Vale investigating a disappearance case. The character is intentionally understated to allow player projection.

Concept art was created to explore psychological tone, posture, and emotional restraint. The design reflects fatigue, analytical distance, and controlled demeanor.

Narrative elements are revealed indirectly through investigation rather than explicit backstory exposition.

Development Insights

Developing DISSOCIATED strengthened my understanding of modular Blueprint architecture, state management in Unreal Engine, and struct-driven system design.

A primary focus was ensuring scalability - new items, threads, or environmental changes can be added without restructuring the core logic framework.

Future improvements would include:

  • Persistent save/load system with serialized thread states.
  • Expanded branching deduction paths.
  • Performance profiling and optimization for larger environments.
  • Enhanced environmental feedback systems.

This project reflects both creative direction and technical system architecture, combining narrative design with structured gameplay engineering.
